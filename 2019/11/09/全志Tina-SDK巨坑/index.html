全志Tina SDK巨坑

这个芯片的SDK着实是令人生草，用Android的Toolchain编译被精简过的OpenWRT，而且document有一些谬误需要修正简直令人绝望

首先F1C200s我是基于Widora的Tiny200开发板做的测试，串口采用CP2102，UART0.
在测试过程中按照Tina SDK Document用pack -d命令打包的固件始终不能正常启动，直到我发现pack可以不带任何arg就可以打包
然后就是在配置文件中有一个debug目录里面塞了一堆awk脚本，这里会把你的串口重定向到PF gourp导致固件无法正常进行串口通信
还有就是配置文件里有很多需要调整的内容，这个就看情况改就好了，和最终设置有关
